Learning Objectives

Overall objective: help you understand how to implement and use a basic form of synchronization primitive: the blocking mutual exclusion lock, commonly referred to as a mutex.

Specific objectives:

  • Learn how to prevent race conditions in a uniprocessor OS kernel by disabling interrupts.
  • Learn how to implement the lock and unlock methods of a mutex on a uniprocessor.
  • Learn how to use a mutex in a concurrent program in order to prevent race conditions (note: this is different to specific objective 1!).
  • Learn how to test and debug a concurrent program.
  • Learn how to recognize and fix race conditions, deadlocks and starvation in concurrent programs.
  • Learn how to use and implement semaphores in a uniprocessor OS kernel.
  • Learn how to use mutex locks in combination with semaphores to solve the producer-consumer concurrent programming problem.
  • Learn how to use and implement monitors and condition variables (with MESA semantics) in a uniprocessor OS kernel.
  • Learn how to solve the dining philosopher's concurrent programming problem using mutex locks and condition variables.
  • Understand the difference between semaphores and condition variables, both in their usage semantics and their underlying implementation.

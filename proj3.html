Assignment 3

Due at start of class on 2019-02-19

Learning Objectives

Overall objective: solidify your understanding of concurrent programming using mutex locks, semaphores, condition variables, and the monitor approach.

Specific objectives:

  • Practice writing your own testing code that generates highly concurrent, randomized tests, and increases the probability of uncovering race conditions and other concurrency-related bugs. The code provided for you in assignment 2 provides many hints on how to approach this!
  • Learn how to generate output that makes it easy to spot race conditions, deadlocks and starvation by simple observation of output. The code provided for you in assignment 2 provides many hints on how to approach this!
  • Develope a solution to the concurrent Gaming Parlor problem, which is a prototype pattern for generic concurrent resource managers in OS kernels.
  • Practice writing concurrent code using the monitor approach with condition variables.

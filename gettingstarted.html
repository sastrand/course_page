Getting Started with BLITZ

This guide is intended to supplement the documentation and files provided with Project 1 of Harry Porter's BLITZ system.

The Easy Way

The BLITZ tool suite is precompiled and ready to run on the CS department's Ubuntu machines. All you need to do is log in to ada.cs.pdx.edu using an ssh client (e.g. ssh for Linux/OSX, PuTTY for Windows, serverauditor for iOS, or JuiceSSH for Android), then add the following lines to ~/.bash_profile:

  export PATH=$PATH:~harry/public_html/Blitz/BlitzBin/Ubuntu64

Log out and back in, and you should now be able to run the BLITZ tools. To test this, type:

  kpl

by itself on a command line. You should see the following output:

  *****  ERROR: Missing package name on command line

  **********  1 error detected!  **********

If you see anything else, then you have not properly configured your PATH variable, or you are not logged in to the right server (you must be logged in to ada.cs.pdx.edu).

This is the easiest way to get started with BLITZ. However, if you would prefer to work locally, then consult the appropriate guide below. Consider this option as a fall-back in case you have trouble getting it running.

Mac OS X (x86, 64 bit)

If you're running Mac OS X on a modern Intel-based machine, then the compiled binaries are already provided for you.

Download the file blitz_mac_x86_bin.tar.gz from the D2L (d2l.pdx.edu). It is saved in the BlitzTools Module on the D2L CS510 course page. Open a Terminal and run the following commands: 


mv /Users/yourusername/Downloads/blitz_mac_x86_bin.tar.gz  ~
cd ~
tar -xvf blitz_mac_x86_bin.tar.gz
mv blitz_mac_x86_bin BlitzTools

Now, follow the instructions below in "Adding the BLITZ tools to your PATH".

Windows (x86, 32 or 64 bit)

To get BLITZ running on Windows, you'll need a copy of Cygwin.

You must install the 32-bit version of Cygwin. The 64-bit version will not work. If you have the 64-bit version already installed, you can install the 32-bit version alongside it.

When installing Cygwin, you should be presented with a screen to select additional packages. You'll need the following:

  • Under "Devel":
    • gcc-g++
    • gcc-core
    • make
  • Under "Utils":
    • util-linux
  • Under "Editors":
    • vim
    • emacs

Now we need to download and compile the source files. By default, cygwin installs in C:\cygwin. If you have chosen a different directory, substitute that here. Right-click on this link and save it to the directory C:\cygwin\home\yourusername\blitz

Now open cygwin and run the following commands:

  cd ~/blitz
  tar -xvf blitz_x86_src.tar.gz
  make

You should now be able to see the following executable files in your ~/blitz directory:

  asm.exe
  blitz.exe
  check.exe
  diskUtil.exe
  dumpObj.exe
  endian.exe
  hexdump.exe
  kpl.exe
  lddd.exe

Now, follow the instructions below in "Adding the BLITZ tools to your PATH".

Linux (x86, 32 or 64 bit)

To install BLITZ on a Linux system, you'll need to compile the binaries from source.

Since BLITZ was designed for a 32-bit architecture, you will need 32-bit versions of the C and C++ libraries in order to compile the code on a 64-bit machine. To install these libraries, run the command appropriate for your distribution (or skip this step if you are using a 32-bit machine):

On Debian-type distros (Debian, Ubuntu, Mint, etc):

First determine which version of g++/gcc you have installed with:

  gcc -v

At the bottom of the output you should see something like:

  Thread model: posix
  gcc version 4.9.4 (Ubuntu 4.9.4-2ubuntu~14.04.1)

Use the first two numbers to install the appropriate 32-bit compilers. In my example the version is 4.9 and the installation would be:

  sudo apt-get install gcc-4.9-multilib
  sudo apt-get install g++-4.9-multilib

On Arch, edit /etc/pacman.conf and uncomment the lines around [multilib], then run:

  sudo pacman -Sy
  sudo pacman -S gcc-multilib

Once you have the appropriate libraries, download the Blitz source code from the link below and compile it by typing make: 

 http://web.cecs.pdx.edu/~harry/Blitz/BlitzSrc/

You should now be able to see the following executable files in your ~/blitz directory:

  asm
  blitz
  check
  diskUtil
  dumpObj
  endian
  hexdump
  kpl
  lddd

Now skip down and read the last section, "Adding the BLITZ tools to your PATH".

Something Else?

This document represents the systems for which I've managed to make a working version of the BLITZ tools. There are instructions for PowerPC and SPARC systems in the BLITZ documentation, but I have not tested them. If you have one of these, or another kind of system (ARM, etc.) and working remotely is not an option for you (if it is, see "The Easy Way" above), send an email to the class mailing list and I'll work with you to get it working.

Adding the BLITZ tools to your PATH

Note: this section only applies if you are working on your local machine. If you are working remotely on chandra.cs.pdx.edu, see "The Easy Way" above.

In order to run these commands from anywhere, you must add ~/blitz to your PATH variable. This can be done by editing your ~/.bash_profile file directly, or by executing the following command:

  echo 'export PATH=$PATH:~/blitz' >> ~/.bash_profile

Then restart your terminal window, or type:

  source ~/.bash_profile

You should now be able to run the BLITZ tools. To test this, type:

  kpl

by itself on a command line. You should see the following output:

  *****  ERROR: Missing package name on command line

  **********  1 error detected!  **********

If you see anything else, then either the program did not compile correctly, or you have not properly configured your PATH variable. Retrace your steps and double check for any mistakes.

If you still can't get it working, send an email to the class mailing list asking for help.
